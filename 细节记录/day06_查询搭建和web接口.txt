
1. sse重点理解
   sse的作用: 后端单向前端推送数据! 本次: 查询以后,流式的结果推送
   sse使用模式:
      基本使用  前后 EventSource  onMessage  == 后端 StreamingResponse
      传入参数  前后 EventSource  onMessage  = url / {session_id } = 后端 StreamingResponse  [1 2 3 ]
      异步任务+传递数据   一个接口启动异步任务 ->   [货架子 容器 队列 异步队列]  <-  一个接口负责给前端返回流式数据
      异步任务+传递数据+事件类型  一个接口启动异步任务 ->   [货架子 {字典 event : data}]  <-  一个接口负责给前端返回流式数据 -> event: 事件\n data: 数据\n\n
