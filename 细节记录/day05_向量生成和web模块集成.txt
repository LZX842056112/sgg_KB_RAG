
1. node_bge_embedding节点实现
   作用: chunk的内容生成稠密和稀疏向量,注意批量生成,考虑content增强
   入参: chunks 没有稠密和稀疏向量
   出参: chunks 有稠密和稀疏向量
   宏观步骤:
      1.日志和任务添加
      2.数据校验chunks
      3.进行批量的向量生成
         state[chunks] = chunks
      4.日志和任务处理
   细化步骤:
      数据校验chunks
        参数: state -> chunks
        返回: chunks
        1.获取state chunks
        2.非空检验,空 -> 抛出异常
      进行批量的向量生成
        参数: 校验好的chunks
        返回: chunks -> 有向量
        1. 准备一个列表,装chunk(有向量的)
        2. 开启循环(+5)  index range(0,total,5)
        3. 截取5个chunk  chunks[index:index+5]
        4. 定一个列表,存储每个chunk要生成的向量字符串
        5. 嵌套循环处理每个chunk  主体:{item_name},内容:{content} -> 列表
        6. result =  嵌入式模型工具生成向量(定一个列表)
           result = {
                dense:[1,2,...],
                sparse:[1,2,...]
           }
        7.  0 1 2 3 4  = range(len(chunks))
            0 -> 截取5个chunk  -> chunk . copy -> 新
            新 -> 0 -> result
            新 -> 准备一个列表,装chunk(有向量的)
        8. 返回最新的chunks

  3.3 流式处理 / 异步处理

       from fastapi.responses import StreamingResponse
        import asyncio

        async def generate_stream():
            # 模拟流式输出（逐字返回）
            words = ["你", "好", "，", "这", "是", "流", "式", "响", "应"]
            for word in words:
                await asyncio.sleep(0.5)
                yield word.encode("utf-8")  # 流式输出需返回字节流

        @app.get("/stream")
        async def stream_response():
            return StreamingResponse(generate_stream(), media_type="text/event-stream")
